# 安全

## 端到端加密

所有消息在离设备前经 Fengni 协议加密，传输层使用 AES-256-GCM。全程无明文——即使是 MQTT 代理也只能看到密文。

## 防流量分析

`paddingMax` 配置项随机化数据包长度，使攻击者无法通过包大小推断消息内容或用户行为模式。

## 设备身份与隔离

- **`deviceId`**：唯一标识每台设备，确保消息来源可追溯
- **`customerId`**：隔离同步组，未授权设备无法接入其他组的通信

## 密钥管理

- `fengni.key` 仅存储在设备本地
- 密钥不同步、不上传至任何服务器
- 密钥丢失不可恢复——需重新部署并生成新密钥对
